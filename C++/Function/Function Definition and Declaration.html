Function Declaration and Definition

Concept of Functions

A function is a block of code designed to perform a specific task. Functions improve readability, reusability, and modularity of programs. In C++, every program must have at least one function: main().

Function Declaration (Prototype)

Theory: A function declaration tells the compiler the function’s name, return type, and parameters, but does not contain the body (implementation). It is also called a function prototype. Syntax: 
return_type function_name(parameter_list);
Declaration allows the function to be used before its definition. 


// Function Declaration (prototype)
#include <iostream>
using namespace std;

// declaration
int add(int a, int b);

int main() {
    cout << "Sum = " << add(5, 10) << endl;
    return 0;
}

// definition (later in program)
int add(int a, int b) {
    return a + b;
}

Function Definition

Theory: A function definition provides the actual body (implementation) of the function. It specifies what the function does when called. Syntax: 
return_type function_name(parameter_list) {
    // body
}


// Function Definition Example
#include <iostream>
using namespace std;

// definition
void greet() {
    cout << "Hello, welcome to C++ Functions!" << endl;
}

int main() {
    greet(); // function call
    return 0;
}

Function Call

Theory: A function call executes the function body. Arguments provided in the call are copied (or referenced) into the function’s parameters. 


// Function Call Example
#include <iostream>
using namespace std;

int square(int x) {   // definition
    return x * x;
}

int main() {
    int n = 6;
    cout << "Square of " << n << " = " << square(n) << endl; // call
    return 0;
}

Key Differences

- Declaration: Only informs the compiler about the function (no body).
- Definition: Provides actual implementation of the function.
- A function must be defined exactly once, but can be declared many times.
- If the definition comes before usage, a separate declaration is optional.

Notes

- Function declaration ends with a semicolon ;, but function definition does not.
- Parameters in declaration must match those in definition (both type and order).
- Functions improve code reusability and structure.
