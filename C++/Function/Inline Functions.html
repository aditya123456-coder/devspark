Inline Functions

Concept

- An inline function is a function where the compiler attempts to expand the function body at the point of call, instead of performing a normal function call (jump in execution).
- This avoids the overhead of function calls, making execution faster for small functions.
- Declared using the keyword inline before the function definition.
- Typically used for short, frequently used functions like getters, setters, or simple calculations.

Basic Example


// inline function example
#include <iostream>
using namespace std;

inline int square(int x) {
    return x * x;
}

int main() {
    cout << "Square of 5 = " << square(5) << endl;
    cout << "Square of 10 = " << square(10) << endl;
    return 0;
}

How Inline Works

- Normally, a function call involves pushing arguments on the stack, jumping to the function code, and then returning.
- With inline, the compiler replaces the call with the actual function code (like a macro, but safer).
- Example: 

square(5);  
// expanded into
5 * 5;
- This eliminates call overhead but can increase code size if overused.

Inline with Default Arguments


// inline with default arguments
#include <iostream>
using namespace std;

inline int add(int a, int b = 10) {
    return a + b;
}

int main() {
    cout << add(5) << endl;    // 5 + 10 = 15
    cout << add(5, 20) << endl; // 25
    return 0;
}

Limitations & Rules

- Inline is a request, not a command â€” the compiler may ignore it.
- Functions with complex logic, recursion, loops, or static variables are usually not inlined.
- Large inline functions can increase binary size (code bloat).
- Inline functions must be defined in the same translation unit or header file to avoid multiple definition errors.
- Inline functions are better than macros since they respect type checking and scope.

Inline vs Macro

- Macros are simple text substitutions, while inline functions are real functions with type safety.
- Example: 


// Macro vs Inline
#include <iostream>
using namespace std;

#define SQUARE(x) (x * x)  // macro

inline int square(int x) { return x * x; } // inline

int main() {
    cout << "Macro SQUARE(5+1) = " << SQUARE(5+1) << endl;  // expands to (5+1*5+1) = 11 (wrong!)
    cout << "Inline square(5+1) = " << square(5+1) << endl; // evaluates correctly = 36
    return 0;
}

Best Practices

- Use inline for small, frequently used functions.
- Place inline function definitions in header files if shared across multiple source files.
- Avoid inlining very large or recursive functions.
- Prefer constexpr for compile-time constant expressions in modern C++.
