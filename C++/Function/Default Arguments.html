Default Arguments

Concept

- A default argument is a value provided in the function declaration/definition that is used if the caller does not supply a value for that parameter.
- They allow functions to be called with fewer arguments while keeping flexibility.
- Default values are assigned from right to left (trailing parameters).
- Syntax: return_type function_name(type param = default_value, ...);

Basic Example


// default arguments example
#include <iostream>
using namespace std;

// function with default arguments
int add(int a, int b = 5, int c = 10) {
    return a + b + c;
}

int main() {
    cout << add(2) << endl;          // uses default b=5, c=10 → 17
    cout << add(2, 3) << endl;       // uses default c=10 → 15
    cout << add(2, 3, 4) << endl;    // all provided → 9
    return 0;
}

Important Rules

- Default arguments must be given from rightmost parameter towards left.
- You cannot “skip” a middle argument when calling — values are matched in order.
- Default arguments are specified only once (usually in the declaration, not both declaration and definition).
- Overloading and default arguments can sometimes conflict — avoid ambiguity.

Declaration vs Definition

Best practice: give default values in the declaration (prototype), not in both places. 


// default args in declaration
#include <iostream>
using namespace std;

int multiply(int a, int b = 2);  // default value here

int main() {
    cout << multiply(5) << endl;   // uses b = 2 → 10
    cout << multiply(5, 4) << endl; // uses b = 4 → 20
    return 0;
}

// definition without default
int multiply(int a, int b) {
    return a * b;
}

Mixing with Overloading

- Sometimes overloading can achieve similar results to default arguments.
- Be careful: combining both can lead to ambiguity errors. 


// default arguments vs overloading
#include <iostream>
using namespace std;

void print(string msg, int times = 1) {
    for (int i = 0; i < times; i++)
        cout << msg << endl;
}

int main() {
    print("Hello");        // default times=1
    print("Hi", 3);        // times=3
    return 0;
}

Default Arguments with Multiple Parameters

- Defaults can be provided for several parameters, but only trailing ones.
- Example: void f(int a, int b = 2, int c = 3);
- Calls: f(1), f(1,4), f(1,4,5) all valid. 


// multiple default arguments
#include <iostream>
using namespace std;

void show(int a, int b = 2, int c = 3) {
    cout << "a=" << a << ", b=" << b << ", c=" << c << endl;
}

int main() {
    show(1);          // a=1, b=2, c=3
    show(1, 5);       // a=1, b=5, c=3
    show(1, 5, 7);    // a=1, b=5, c=7
    return 0;
}

Notes & Best Practices

- Use default arguments to reduce code duplication.
- Avoid writing conflicting overloads with default arguments.
- Place defaults in the function declaration (header file) not in the definition (cpp file).
- For readability, use defaults only when they make logical sense (e.g., default size, default tolerance).
