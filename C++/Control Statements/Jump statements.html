Jump Statements

break Statement

Theory: The break statement is used to immediately exit from a loop or a switch block. Control jumps to the first statement after the loop or switch. 


// break statement - C++
#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 10; i++) {
        if (i == 5) {
            break; // exits loop when i == 5
        }
        cout << i << " ";
    }
    return 0;
}

continue Statement

Theory: The continue statement skips the rest of the code inside the current loop iteration and moves to the next iteration of the loop. 


// continue statement - C++
#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 10; i++) {
        if (i % 2 == 0) {
            continue; // skip even numbers
        }
        cout << i << " ";
    }
    return 0;
}

goto Statement

Theory: The goto statement transfers control unconditionally to a labeled statement in the same function. It can make code harder to follow, so its use is discouraged in modern C++. 


// goto statement - C++
#include <iostream>
using namespace std;

int main() {
    int n = 1;

start:
    cout << "Value of n = " << n << endl;
    n++;
    if (n <= 5) {
        goto start; // jump back to label
    }
    return 0;
}

return Statement

Theory: The return statement ends the execution of a function and optionally sends a value back to the caller. In main(), returning 0 usually indicates successful program termination. 


// return statement - C++
#include <iostream>
using namespace std;

int add(int a, int b) {
    return a + b; // return value to caller
}

int main() {
    int result = add(10, 20);
    cout << "Sum = " << result << endl;

    return 0; // end of program
}

Notes

- Use break to exit loops/switch when a condition is met.
- Use continue to skip current iteration but keep looping.
- Avoid goto unless absolutely necessary (can reduce readability).
- Always use return in functions with non-void return type to return a proper value.
